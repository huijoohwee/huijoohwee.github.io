# AI Builders/Agent Forge Hackathon - Build OpenClaw AI Assistant

## OpenClaw AI Assistant 

The era of simple chatbots is over. 
Real AI applications need to access live data, write their own code, autonomously manage their own state, and run reliably in the cloud.

The Building OpenClaw AI Assistants hackathon gives you the ultimate stack to ship production-ready agents in one day. 
Assemble a team of one to six people — or go solo — and build, test, and deploy a working OpenClaw solution together. 
We are moving beyond local prototypes. 
If it doesn't **deploy and manage real data**, it doesn't count.

---

## Team Format

Here are some directions your team could take:

- A custom UI to manage multiple OpenClaw Agent Forge instances build a unified dashboard for orchestrating and monitoring several agents at once.

- Custom tools or skills for a single OpenClaw agent — extend what one agent can do by shipping new capabilities it can call on.

- A collaborative, cross-role project powered by OpenClaw — combine business development, design, and engineering: 
one teammate builds a pitch deck, another designs the UI, and a third wires it all together through OpenClaw.

You're free to come up with your own idea. 
The only requirement: your project must be **built on OpenClaw** and **deployed live by demo time**.

---

## The Challenge

Build an autonomous OpenClaw assistant or data-driven application that solves a real-world problem.

Unlike standard hackathons where you build UI shells, this event focuses on functionality, live data, and true autonomy. 

Your OpenClaw agent should be able to gather intelligence (Scraping), autonomously spin up and manage its own database (Storage), process the information, and live on the web (Deployment).

---

## The Stack (Your Arsenal)

You will have access to premium credits and tools from our innovation partners to build your OpenClaw solution. 

Follow this workflow to assemble your production-ready AI assistant:

Step 1 — [Zeabur](https://zeabur.com) (The Launchpad): 
Start by deploying your OpenClaw instance to the cloud. 
Ship your backend, APIs, and web interfaces with a single click, ensuring your project is live, accessible, and scalable from the start — no DevOps friction required.

Step 2 — [ZenMux/Agnes AI](https://zenmux.ai/sapiens-ai/agnes-1.5-pro) (The Brains): 
Install the purpose-built Agnes-Claw or Agnes-Claw-Pro models as your primary LLM backend. 
They deliver high-quality, low-cost reasoning and advanced tool-calling capabilities specifically optimized for powering the decision-making loops of OpenClaw agents.

Step 3 — [Bright Data](https://brightdata.com/blog/ai/openclaw-with-bright-data) (The Eyes): 
Give your assistant the ability to browse the web. 
Install Bright Data OpenClaw skills to enable web search capabilities, allowing your agent to fetch real-time data, monitor trends, or aggregate research automatically using reliable, large-scale infrastructure.

Step 4 — [TiDB Cloud Zero](https://zero.tidbcloud.com) (The State): 
Add structured data persistence to your agent. With zero sign-up, zero config, and zero friction, your OpenClaw agents can provision fully functional, disposable distributed SQL databases using just a single line of curl to effortlessly and autonomously manage their own state.

Step 5 — [mem9](https://mem9.ai) (The Memory): 
Equip your OpenClaw assistants with long-term memory and contextual awareness. 
Use mem9 to seamlessly store, embed, and retrieve past interactions and user preferences, allowing your OpenClaw agents to remember who you are and retain context across infinite sessions.

Step 6 — Telegram Bot Father
A hub for creating bots in telegram, used as the interface for interacting with your AI assistant in real-time.

## Example Project Ideas

- ​Market Watcher: An OpenClaw agent that scrapes competitor pricing across 50 e-commerce sites, provisions a TiDB Cloud Zero database to store historical pricing, and adjusts your own pricing database via API.

- ​News Synth: An OpenClaw app that aggregates breaking news from specific niche forums, indexes the content into an agent-provisioned database, and summarizes them into a daily briefing.

- ​Real Estate Hunter: An OpenClaw agent that monitors listing sites for specific criteria, stores historical trends in its own SQL database, and alerts users instantly when a deal appears.
